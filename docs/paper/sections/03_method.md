## 3.1 Setup

A `sample_instance` is one (target_image, question, anchor_draw) triplet.
For each `sample_instance` we evaluate up to four conditions:

| short | code label | second image |
|---|---|---|
| `b` | `target_only` | none (baseline) |
| `a` | `target_plus_irrelevant_number(_S?)` | image with one digit (the anchor) |
| `m` | `target_plus_irrelevant_number_masked(_S?)` | same anchor image with the digit pixel region inpainted out |
| `d` | `target_plus_irrelevant_neutral` | digit-free FLUX-rendered image (2-image-distraction control) |

Each condition produces a model prediction:
`pred_b / pred_a / pred_m / pred_d`. The ground truth is `gt`; the
anchor's numerical value is `anchor`.

Three condition gaps are interpretable:

- (a âˆ’ d) separates anchoring from generic 2-image distraction.
- (a âˆ’ m) isolates the digit-pixel contribution from anchor-scene
  background distraction.
- ((a, base-wrong) âˆ’ (a, base-correct)) maps uncertainty modulation
  (Phase-A; refined to a continuous quartile in Â§6).

## 3.2 Stimulus inventory

- **Target images**: per-dataset (Â§4).
- **Anchor inventory**: 128 rendered-digit images generated by
  FLUX.1-schnell (1024 Ã— 1024, 1 inference step, guidance scale 0,
  filename = digit value; e.g. `5.png`). Each contains a single Arabic
  numeral on a generated scene.
- **Mask inventory** (`m`): each anchor image with the digit pixel
  region replaced by a Telea inpaint (OpenCV). The mask region is the
  dilated bounding box of the digit, detected via PaddleOCR with a
  synthetic-bbox fallback. Background scene is preserved. OCR-validated:
  no detectable digit post-inpaint.
- **Neutral inventory** (`d`): digit-free FLUX renders with matched
  scene-stylistic distribution.

## 3.4 Canonical metrics (M2, C-form)

Let `pa = pred_a` (anchor-condition prediction), `pb = pred_b`
(baseline prediction), `anchor` (anchor's value), `gt` (ground truth).

```
adopt_rate            = #(pa == anchor AND pb != anchor) / #(pb != anchor)
direction_follow_rate = #( (pa - pb) Â· (anchor - pb) > 0  AND  pa != pb )
                        / #(numeric pair AND anchor present)
exact_match           = #(pa == gt) / #(numeric pair)
anchor_effect_M       = M(a-arm) - M(d-arm)
```

Same form on the m-arm (substitute `pred_m` for `pa`).

The `direction_follow_rate` numerator measures whether `pa` shifted
**from the baseline `pb` toward the anchor side of `pb`** â€” anchor pull
as a baseline-relative shift. Using `pb` (not `gt`) as the reference
makes the metric depend only on model outputs and the anchor draw, not
on whether the model happened to be correct in the baseline condition,
and not on `gt` itself. This is robust to per-question stimulus
variability and to ground-truth distribution differences across
datasets.

The `pa != pb` clause is structurally redundant under the C-form
(when `pa == pb`, the factor `(pa - pb) = 0` zeros the numerator) but
is kept explicit for clarity and as a hardener against parsing
edge cases.

## 3.5 Per-row flags persisted to predictions.jsonl

Six flags are written per record so downstream summarisation can be
verified and re-derived from the raw predictions:

- `anchor_adopted` (M1 paired numerator)
- `anchor_direction_followed` (C-form sign-based)
- `anchor_direction_followed_moved` (= sign-based AND `pa != pb`)
- `pred_b_equal_anchor` (paired-denominator gating)
- `pred_diff_from_base` (movement gating)
- `numeric_distance_to_anchor`

A regression test enforces driver-row schema parity, so the gap that
caused our 2026-04-28 audit (`summarize_condition` reading missing
keys as 0) cannot recur silently.

## 3.6 Decision rationale for the C-form

The C-form `(pa - pb) Â· (anchor - pb) > 0` was chosen over two
alternatives that surfaced during method development:

- **anchorÂ·gt form** `(pa - gt) Â· (anchor - gt) > 0` (the M1-era code
  body) â€” measures whether `pa` and `anchor` are on the same side of
  `gt`. The metric depends on the anchor draw via `anchor âˆ’ gt`, so
  per-record indicators vary with stimulus rather than model
  behaviour. On datasets where `anchor` and `gt` are similarly
  bounded (MathVista) the two forms produce nearly identical numbers,
  but on VQAv2 where `gt âˆˆ {0..8}` and `anchor âˆˆ {0..9}` overlap
  substantially the dependency on stimulus matters. Discarded.
- **pbÂ·gt form** `(pb - gt) Â· (pa - gt) > 0` â€” measures whether `pa`
  and `pb` are on the same side of `gt` (anchor-agnostic). The anchor
  does not appear in the formula, so the metric measures
  pb-invariance / stickiness rather than anchor pull, conflicting with
  the metric's name. Discarded.
