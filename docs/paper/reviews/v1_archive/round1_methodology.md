## Summary

The paper presents a coherent, ambitious story (graded cross-modal anchoring, multi-layer-redundant mechanism, deployable subspace projection, capability-preserving) and the *headline* mitigation numbers (E6 §6.2.3 Table 6 and E8 §7 Table 8) are reproducible to ≤0.05 pp against canonical CSVs. **However**, methodology rigor in several mid-paper tables is markedly weaker than in the headline: §4.5 Table E.2 mode summary is unverifiable on the OneVision row by the authors' own E1d evidence file (analyzer bug, 0.000 baseline), §5.1 Table 5 (E4) has em values that **do not match** the canonical `E4-mitigation-evidence.md` (LLaVA/ConvLLaVA appear **swapped or fabricated**), §4.3 Table 3 **TallyQA gemma3-27b row is wrong** (0.138/0.117 should be 0.074/0.053 per `E5c_per_cell.csv`), the abstract contains a basic arithmetic error ("5 dataset × 7 model = 85 cells", since 5×7 = 35), and several CIs are reported without the bootstrap method anywhere being described. The §6.5 Table 7 baseline-comparison "+57 %" ActAdd / "+56 %" LEACE backfire numbers do not trace to any canonical run table — only +56 % LEACE is sourced (from a tracker MD), the +57 % ActAdd value appears nowhere I could find. Strong story, table-level discipline needs a pass before submission.

## Strengths

- **Mitigation headline tables are clean.** Table 6 (E6 5-dataset paired-sids) matches `_data/stage4_final_per_dataset.md` to ≤0.05 pp on every cell (e.g., paper InfoVQA Δ adopt(a) +1.0 pp ↔ canonical +0.0095). Table 8 (E8 capability) matches `_data/capability_eval_per_benchmark.md` exactly on all 6 benchmarks + macro Δ +0.41 pp + verdict `STRICT_FREE_LUNCH`. The mitigation chain that the paper hangs its EMNLP-Main argument on is internally consistent.
- **γ-β arithmetic checks out.** Paper Table 4 (Qwen3-VL Instruct vs Thinking) reports wrong/correct df(a) cells matching `_data/experiment_e5e_mathvista_reasoning_per_cell.csv` exactly (Instruct wrong 0.256, correct 0.021; Thinking wrong 0.327, correct 0.267). The all-base ratios (×1.6, ×2.9, ×12.7) reproduce when re-derived from the per-cell counts (n_pb_ne_anchor / n_numeric_anchor weighted means).
- **The (a − m) contrast design rationale (§6.2.1 Insight) is the strongest argumentative move.** It explicitly ties the §4.3 (b, m, d) controls to the E6 SVD setup and explains why other choices ((a−b), (a−d)) are inferior. This is genuine mechanism-to-mitigation traceability rather than post-hoc rationalisation.
- **§6.5 Table 7 is the right framing.** A multi-method comparison table where the chosen method passes the *strict free-lunch* bar that the paper itself defines is the right way to argue for a mitigation contribution. The framing "single-direction failure mode + weight-space failure mode → multi-direction residual the only candidate" is methodologically tight (modulo §6.5's traceability gap, see [CRIT-3]).
- **§5.3 honestly surfaces the OneVision dataset-dependent peak**, which is a structural threat to the §6.2 single-(L,K,α) generalisation pitch. Acknowledging this *before* reporting the 5/5 generalisation result builds credibility.

## Weaknesses (ordered by severity)

- **[CRIT-1]** **§5.1 Table 5 (E4 Phase 2) Δem column does not match canonical.** Paper rows: LLaVA-1.5 +1.30 pp / ConvLLaVA +0.85 pp / InternVL3 +0.49 pp. `docs/insights/E4-mitigation-evidence.md` Phase 2 headline (88,650 records full validation, the source the paper cites) reports: LLaVA-1.5 +0.77 pp / ConvLLaVA +1.30 pp / InternVL3 +0.49 pp. The +1.30 has been moved from ConvLLaVA to LLaVA-1.5, +0.77 has been *dropped* from LLaVA-1.5, and ConvLLaVA's +0.85 **does not appear in any canonical table I could find** (closest: Phase 1 stratified n=200 sweep gives ConvLLaVA +0.0 pp). The headline range "0.49–1.30 pp" the paper cites in §1.3 / abstract is correct as a *range*, but the per-row table values are wrong. Since this is the only mid-stack-cluster mitigation evidence and the "complementary E4/E6" framing rests on it, this row-level error is paper-blocking.
- **[CRIT-2]** **§4.3 Table 3 TallyQA gemma3-27b row has the wrong absolute values (gap is right by accident).** Paper: adopt(a)=0.138, adopt(m)=0.117, gap +2.1 pp. Canonical `_data/E5c_per_cell.csv` (`gemma3-27b-it,TallyQA,wrong,anchor,S1`) gives adopt_cond=0.0737, adopt_cond_masked=0.0532, gap +2.05 pp. The 0.138/0.117 numbers look like a hand-copy mix-up — 0.138 is gemma3-27b's *VQAv2* adopt(a) value (one row above in the same table) and 0.117 has no source I could locate. This is a transcription error that will be caught by any careful reader cross-checking `headline-numbers.md §E5c`. Same row on llava-interleave: paper +2.5 pp, canonical +2.6 pp (rounding only — not the same severity).
- **[CRIT-3]** **§6.5 Table 7 ActAdd "ChartQA backfire +57 %" and LEACE "+56 %" are not jointly traceable.** I located the +56 % LEACE figure in `docs/experiments/E6-tally-only-rerun-tracker.md:480` ("LEACE | Cross-dataset failed even on gt ∈ [0,8] (ChartQA backfires +56 %)"). I could not locate **+57 %** for ActAdd in any `docs/insights/`, `docs/insights/_data/`, or `docs/experiments/E6-*.md` source. Closest is `E6-steering-vector.md` reporting "α=4 / wrong-v at L24" backfire +14.0 % df, and various per-bin backfires +11.1 % to +96 % on ChartQA gt-bins for *Subspace* method (not ActAdd). Either (a) +57 % refers to an aggregation that needs to be surfaced and cited, or (b) it is wrong. Given the paper hangs its "single-direction fails" mechanism-prediction on this number, the source must be pinned to a specific run + aggregation rule.
- **[CRIT-4]** **Abstract arithmetic error: "5 dataset × 7 model = 85 cells".** 5 × 7 = 35, not 85. Canonical `_data/L1_proxy_monotonicity.csv` filtered to `cond_class=a, proxy=cross_entropy` has 85 rows decomposed by (experiment × dataset × model × stratum) — i.e., the 85-count *includes the S1–S5 stratum split*, plus a 6th dataset (the L1 panel actually covers 6 datasets: VQAv2, TallyQA, ChartQA, MathVista, PlotQA, InfoVQA, not 5). Per `L1-confidence-modulation-evidence.md` head: "85 anchor cells across {6 datasets} × {7 models}". The "5 × 7 = 85" is wrong on both factors and the multiplication. The body §4.5 has the same structural error ("5 dataset × 7 model panel의 85개 anchor cell").
- **[MAJOR-5]** **§5.2 "Single-layer mask ablation은 6/6 모델에서 null" is misleading; OneVision is not actually evaluated cleanly.** `docs/insights/E1d-causal-evidence.md` 2026-05-04 update explicitly states: "OneVision aggregator output shows 0.000 baseline df on all 4 datasets — the analyzer's stratification logic does not match OneVision's per-dataset susceptibility CSV format. Raw predictions are correct in `outputs/causal_ablation/llava-onevision-qwen2-7b-ov/<run>/predictions.jsonl`. Refining the OneVision-aware analyzer is a Phase 3 follow-up." The 6/6 panel claim therefore really means "5/5 mech panel models null + OneVision pending analyzer fix". The abstract and §5.2 should state "5 mech panel + OneVision pending" rather than "6/6". The downstream rhetorical move ("multi-layer redundancy *predicts* single-direction failure") is unaffected, but the cardinality claim is currently overstated.
- **[MAJOR-6]** **Bootstrap CI methodology is never specified.** §7 Table 8 reports "95 % CI [+1.14, +3.28]" (HallusionBench) and "[−0.21, +0.09]" (POPE) — no description anywhere of bootstrap method (paired vs cluster, n_resamples, percentile vs BCa, what unit gets resampled). `E8-capability-preservation-evidence.md` also does not describe the method. Paper §7 has the *only* statistical-significance claim in the entire body ("excludes zero", "pinned to zero") and that claim rests on an unspecified procedure. §3.2 / §4 / §6.2.3 do not report CIs at all even where paired denominators are tiny (n=170 MathVista, n=224 ChartQA in §6.2.3) — see [MAJOR-9].
- **[MAJOR-7]** **§5.1 Insight 2 "5/5 main matrix dataset에서 OneVision Main의 peak이 L20-L23에 안정" conflates two different "peaks".** From `_data/cross_dataset_peaks.csv`, OneVision's *answer-step attention peak layer* is L=14 on InfoVQA + VQAv2 and L=27 on PlotQA + TallyQA — i.e., **bimodal on L14/L27, not "stable in L20–L23"**. The L20–L23 cluster is the *digit/anchor concentration peak* in `_data/E1_patch_concentration_per_layer.csv` (OneVision a-arm: L20=0.507, L23=0.492, L25=0.459), an entirely different quantity (per-layer anchor-bbox mass, not per-layer attention peak depth). §5.1 reads as if attention peak and bbox concentration peak are the same; they are not, and §5.3 ("OneVision Main — dataset-dependent peak") in fact contradicts §5.1 explicitly (§5.3 reports L=27 / L=14 dataset-dependent). Either §5.1's claim should be qualified to "calibration-set bbox concentration peak L20–L23" or removed.
- **[MAJOR-8]** **§5.1 "각 모델은 정확히 하나의 peak layer를 가진다 (4 archetype)" is overstated for FastVLM.** `_data/cross_dataset_peaks.csv` rows for fastvlm-7b show answer-step peak L=22 on VQAv2, L=27 on InfoVQA (peak_layer_frac=1.0), L=17 on PlotQA, L=23 on TallyQA. That is **four different peak layers across four datasets**, not "exactly one peak". The "FastVLM late + text-stealing (L22)" archetype label cherry-picks the VQAv2 row. Paper should either present per-dataset peak layers (as `cross_dataset_peaks.csv` does) or qualify the archetype claim to a specific dataset. The same caveat applies more weakly to OneVision (L14/L27 bimodal, not single-peak) — §5.3 *does* surface this for OneVision, but §5.1's "정확히 하나의 peak layer" prose is uncorrected.
- **[MAJOR-9]** **§6.2.3 paired-sids small-denominator cells (n=170 MathVista, n=224 ChartQA) report no CIs.** Paper Table 6 reports "MathVista n=170, Δ df(a) = −4.1 pp, Δ em(b) = +9.4 pp" without a confidence interval. With n=170 paired and the underlying df rate ~0.20, the 95 % bootstrap CI half-width on Δ df is approximately ±0.06 (Wilson-style), so −4.1 pp is plausibly statistically distinguishable from 0 but the paper has not established this. Same applies to ChartQA n=224. Given the §1.6 contribution claim is "5/5 dataset df reduction" — and 2 of the 5 cells have n < 250 — readers will want CIs. This is a routine ARR / NeurIPS reviewer ask.
- **[MAJOR-10]** **§4.3 Insight 3 "PlotQA un-mitigated free-lunch" claim has no statistical test.** Prose: "7개 중 6개 모델이 PlotQA에서 em(a) > em(b) (em delta +0.6 ~ +5.0 pp)." This is presented as an empirical fact with cross-model robustness, but per `_data/main_panel_5dataset_summary.md`, the PlotQA per-model em(a) values relative to em(b) cannot be re-derived from the 5-dataset summary alone (em(b) is acc(b), not em(b) under canonical M2; the comparison against em(a) is not directly tabulated). The +0.6 to +5.0 pp range needs to trace to a specific aggregation. (`docs/insights/E7-plotqa-infovqa-evidence.md` is the candidate source; reviewer should pin to a specific section and table.)
- **[MAJOR-11]** **§4.5 worked-example numbers in Figure 6 caption do not match the canonical CSV.** Caption: "Worked example E5c VQAv2 wrong-base S1 × LLaVA에서 df 0.024 → 0.062 → 0.158 → 0.254 (gap +23 pp)." So Q1 and Q4 match the paper but **Q2 and Q3 are wrong** (paper 0.062 / 0.158 vs canonical 0.044 / 0.137). Also: the canonical row is **all-base, not wrong-base** (no `base_correct` filter on the L1 quartile data). The "wrong-base × LLaVA" label in the caption is inconsistent with the data shape; either re-aggregate the wrong-base subset and report, or relabel to "all-base". The +23 pp gap value (0.254 − 0.024) survives because Q1 and Q4 are correct, but the intermediate quartiles do not.
- **[MAJOR-12]** **§3.3 dataset n claim "ChartQA n=5,390" is misleading.** Per-model effective n in `_data/main_panel_5dataset_summary.md` is 129–517 on ChartQA (e.g., qwen2.5-vl-32b n=129). The paper's "n=5,390" is the *raw integer-GT subset count before stratification × eligibility filtering*. The paper's headline claims (e.g., §6.2.3 ChartQA n=224) are based on the stratified subset, not 5,390. The paper should state both numbers (raw 5,390 / per-cell 129–517) or pick one consistently. Same concern for "TallyQA n≈38k" vs per-model 6,934–14,772 in the main_panel summary, "PlotQA n≈5,000" vs per-model 926–4,610.
- **[MINOR-13]** **§4.6 Insight 3 "acc(b) Thinking 0.587 vs Instruct 0.647" — the "acc(b)" label is misleading.** Per `experiment_e5e_mathvista_reasoning_per_cell.csv`, this is computed from the *d-arm correct fraction* (Instruct: 249/385 = 0.647; Thinking: 226/385 = 0.587). That's `acc(d)`, not `acc(b)`, since the dataset has no separate b-arm baseline run for the γ-β pair. Either rename the metric or document the d-arm proxy.
- **[MINOR-14]** **§4.5 proxy choice rationale missing.** Paper picks "51/85 (60 %, log_prob_sum)" but states "cross_entropy paper-default". Per `_data/L1_proxy_comparison.csv`, cross_entropy gives 43/85 (50.6 %) — log_prob_sum gives 51/85 (60 %). The paper picks the *better* number while citing the *other* proxy as default. - **[MINOR-15]** **§5.1 E1-patch concentration "0.468–0.631" range omits OneVision.** Per `E1-patch-evidence.md` and `_data/E1_patch_concentration_per_layer.csv`, the perfect-square panel range is 0.468 (llava-1.5) to 0.631 (gemma4-e4b) — but adding OneVision (0.507) and qwen2.5-vl-7b (0.540) would give a narrower range 0.468–0.631 (unchanged endpoints) but the panel composition would be 6 not 4 models. - **[MINOR-16]** **Reasoning citation [Bae et al., 2025] is not actually used in §4.6 Insight 3.** Paper claim: "Reasoning trace가 정확도 향상을 보장하지 않을 뿐 아니라 ([Bae et al., 2025]의 Idis 결과와 부합)". ## Specific issues by axis

### A. Sample sizes / statistics

Sample-size reporting is **uneven**:
- §4.1 Table 2 reports "n=17,730 each" — verifiable against `headline-numbers.md §B`, OK.
- §3.3 reports per-dataset n claims (ChartQA n=5,390 etc.) but these are pre-stratification counts, not the n used in tables (see [MAJOR-12]).
- §6.2.3 Table 6 reports n_paired (4978 / 2306 / 443 / 224 / 170) — matches canonical, OK.
- §7 Table 8 reports per-benchmark n — matches canonical, OK.

Statistical reporting is **weak**:
- The only CIs in the paper are §7 Table 8. Bootstrap method is undescribed (see [MAJOR-6]).
- §4.5 monotonicity is reported as "51/85 cells" — a count without a CI. With 85 trials and a per-cell pass rate of ~60 %, a 95 % bootstrap CI on the "fraction monotone" estimate would be approximately [50 %, 70 %]. The paper does not engage with this.
- §6.2.3 Δ values for n=170 / n=224 cells are point-estimates only (see [MAJOR-9]).
- §4.3 Table 3 small denominators (TallyQA wrong-base S1 ~95 per arm per `headline-numbers.md §E5c` noise-floor caveat) are not reported or flagged in-paper.

### B. Baselines

Paper §6.5 Table 7 lists 5 baselines (ActAdd, LEACE, query-adaptive PCA+Ridge, CogBias decode-time, MIA-DPO LoRA). Description quality varies:
- **Insufficient hyperparameters.** No specific α, layer, K, calibration-set descriptions for any of the 5 baselines. Reviewer cannot reproduce. The chosen E6 cell *does* report (L=26, K=8, α=1.0); the 5 baselines do not state what was chosen.
- **"+57 % ActAdd / +56 % LEACE" backfire numbers**: only +56 LEACE traceable (see [CRIT-3]).
- **MIA-DPO LoRA "−5.85 pp on VQAv2"**: traceable to `docs/experiments/E6-tally-only-rerun-tracker.md` Method 3 description ("DPO mix_synthetic ... em −5.85 pp on VQAv2 borderline kill"), which is a single-cell number and does not represent an LR sweep. Should be qualified.
- The "1/4 임계 미달" verdict for QAO and CogBias is opaque — what 4 thresholds, what cells?

Recommendation: For each baseline, append one paragraph in §6.5 with (a) the implementation pointer (script path), (b) the calibration set used, (c) the chosen hyperparameters, (d) the per-cell results (or pointer to a table) backing the verdict.

### C. Ablations

§5.2 ablation table (Appendix E.2) lists 5 modes with summary "6-model 평균 Δdf". This is **insufficient**: the per-mode-per-model matrix is what `outputs/causal_ablation/_summary/per_model_per_mode.csv` actually contains, and `E1d-causal-evidence.md` "What we found" gives per-mode ranges (e.g., upper-half −4.0 to −10.5 pp on **6/6** is the canonical statement, but as [MAJOR-5] notes, OneVision's 0.000 baseline is an analyzer artifact, not a real null). Paper should reproduce the per-model rows in an appendix, not aggregate them.

§6.2.2 "27-cell pilot grid (L ∈ {25, 26, 27} × K ∈ {2, 4, 8} × α ∈ {0.5, 1.0, 2.0})" — selection rule stated ("em-deal-breaker ≤ −6 pp"), but the **per-cell grid results are not in the paper or any cited canonical table I could locate**. The 27-cell heatmap is the standard ablation that backs the L=26/K=8/α=1.0 choice and its absence is a clear gap. (`outputs/e6_steering/llava-onevision-qwen2-7b-ov/sweep_subspace_*` referenced in `stage4_final_per_dataset.md` head should be aggregable into a 27-cell table.)

### D. Confounds

- **§4.5 worked example is one cell.** The paper's headline "Q4-Q1 gap +23 pp" is a single (E5c, VQAv2, llava-interleave, a, S1, cross_entropy) cell — the E5c VQAv2 cell only. Generalisability is claimed via the 85-cell mean +15.6 / +19.1 pp, which is the right move, but readers should be cued that the 23 pp number is one cell, not the panel mean. Also see [MAJOR-11] — this single cell's intermediate quartile values do not match the canonical CSV.
- **§4.6 γ-β is N=1 architecture-pair.** Single Qwen3-VL Instruct vs Thinking. The paper's contribution claim "VLM 최초의 reasoning-amplifies-anchoring" is correct as a *first-evidence* claim, but the prose "텍스트 LRM 문헌의 reasoning-amplifies-bias 현상이 VLM에 일반화" is overclaiming on N=1. Soften to "consistent with text-LRM literature on a single Qwen3-VL pair; cross-architecture generalisation deferred". The §8.2 limitations section mentions paraphrase, closed-weight, human baseline, mid-stack-cluster as limitations but **does not list "γ-β is N=1 reasoning pair"** — should be added.
- **§6.2.3 paired-sids paired correctly (sids parseable on both arms in baseline AND mitigation, per `stage4_final_per_dataset.md` source notes), so the pairing is sound; the small-n confound is statistical, not selection.**
- **§4.3 PlotQA un-mitigated free-lunch is reported as cross-model (6/7) but with no per-model numbers in the body.** The +0.6 to +5.0 pp range is offered in Insight 3 but readers cannot identify which 6/7 nor which is the outlier without going to `E7-plotqa-infovqa-evidence.md`.

### E. Negative results disclosure

- **§5.2 / Appendix E.2 fails to report `ablate_lower_half` heterogeneity.** `E1d-causal-evidence.md` is explicit: "Lower-half ablation: 3/6 BACKFIRE, 1/6 reduce, 2/6 flat", with Gemma +0.27, LLaVA-1.5 +0.165, InternVL3 +0.068 backfires. The paper Appendix E.2 lists `ablate_lower_half` as "~0" and only highlights upper-half. The 3/6 backfire pattern is *exactly the kind of negative result that distinguishes correlational signature from causal locus* and should not be hidden.
- **§5.2 Insight 1 / Insight 3 do not surface ConvLLaVA-vs-LLaVA-1.5 lower-half divergence (ConvLLaVA −0.150 vs LLaVA-1.5 +0.135).** Per `E1d-causal-evidence.md` "Sub-finding flagged as 'do not generalise to single architecture cluster'", this is documented in canonical evidence but not surfaced in §5. It's another high-value methodological caveat that should appear at least as a footnote.
- **§6.2.2 — 27-cell pilot grid: only the chosen cell is shown.** No grid heatmap, no per-cell em(a) / em(b) / Δdf for the other 26 cells. Per [Ablations C], this is a structural gap.
- **§4.7 / Appendix D appropriately surfaces InternVL3 H7 reversal and falsification of B2 / B3 — this is good negative-results discipline, contrast to §5/§6.**

## Open methodological questions for future rounds

- **Are the §6.2.3 paired-sids deltas robust to bootstrap CI on the small n=170 / n=224 cells?** With p ~ 0.20 and n=170, half-width ~6 pp. Δ df = −4.1 pp could be borderline non-significant. Resolution: bootstrap, report.
- **Does the L=26 / K=8 cell lose if the 27-cell pilot is repeated on calibration sets other than PlotQA+InfoVQA pooled?** §6.2.2 only describes one calibration. Section 6.2 Insight 1 implies amplitude-correlated dataset-shared subspace, but that is interpretation, not test. Future round: ablate the calibration set.
- **Is the InternVL3 "confident-confabulation" hypothesis (§4.7 / Appendix D, future-work) testable on cross-encoder data?** `L1-confidence-modulation-evidence.md` §B3 follow-up notes "the reversal coincides with this encoder family — a single-model finding that needs cross-encoder testing to be falsifiable". Adding a second InternViT model would either reproduce the H7 reversal (mechanistic) or break it (idiosyncratic).
- **Does the (a − m) contrast subspace structure hold on §4.5's wrong/correct split?** I.e., if you SVD on (a-arm wrong-base) − (m-arm wrong-base), do you get a different K=8 subspace from (a-arm correct-base) − (m-arm correct-base)? If yes, that confirms the "two effects merged in K=8" reading from §6.3 Insight 1; if no, there's a single confidence-axis-aligned representation. Either way, would tighten the §6.3 narrative.
- **The §6.3 "wrong-base error mode debiased on b-arm" interpretation predicts that other anchor-free hallucination tasks should show similar projection benefits.** §7 HallusionBench +2.21 pp is consistent. Falsifiable prediction: tasks where b-arm wrong-base errors are **not** confidence-low (e.g., counting tasks where the model is wrong but high-confidence) should show *less* projection benefit. Worth measuring on, e.g., TextVQA or ScienceQA in a future round.
