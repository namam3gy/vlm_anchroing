## Decision

**Borderline — leaning reject for top venues (EMNLP/NeurIPS Main), accept for Findings / mid-tier with major revisions.** The headline mitigation chain (E6 §6.2.3 + §7) is reproducible to ≤0.05 pp against canonical CSVs, the (a−m) digit-pixel contrast and 4-clause free-lunch definition are genuinely novel methodological moves, and the multi-layer redundancy → single-direction failure prediction chain is a paradigm-quality finding *if* the predict-then-verify sequencing holds. **But three structural problems block acceptance at top-tier venues:** (1) the paper is a 834-line *experiment changelog*, not a 8-9 page conference paper — the v3–v8 changelog physically embedded at lines 831–833 is dispositive, (2) the §5.4 routing-vs-integration framework is positioned as *prospectively* predicting §6.4 + §4.6, but the dates and source documents reveal §5.4 was authored *after* §6.4 + §4.6 results were in — this is **post-hoc framing of a retrospective synthesis as a predictive theory**, which is the single most consequential methodology violation in the paper, (3) the abstract continues to claim "uniquely passes 4-clause free-lunch" while §6.2.3's own self-correction shows only Δem(b) is multiplicity-robust (Δdf is 1/5 CI-clean — PlotQA only) — the abstract underweights the body's self-disclaimer.

## Severity-graded issues

### CRITICAL (paper cannot be accepted unless fixed)

- **[CRIT-1] §5.4 framework is presented as *prospective prediction* but is a *retrospective synthesis* — methodologically dispositive.** Lines 278–297 (§5.4) frame the routing-vs-integration framework as deriving four predictions that are then "verified" in §6.4 + §4.6. The abstract (line 13) and §1.5 (line 43) describe this as a *"predict-then-verify chain (§5.2 → §5.4 → §6.4)"*. However: (a) §6.4 LEACE rank-1 ChartQA +56 % backfire was observed pre-§5.4 — its primary evidence source `docs/experiments/E6-tally-only-rerun-tracker.md:480` predates the v7 framework writeup (v7 changelog line 831: "Round-5 bar-raiser review 대응" added the framing post-observation). (b) §4.6 γ-β bridge per `gamma-beta-bridge-evidence.md` line 3 is dated "2026-05-10 (v2 with re-calibration rescue)" and was added in v8 (line 833 — "(8) §8.4 (load-bearing follow-up) item 5" describes shifting paired-bootstrap CI into v8 close, dated 2026-05-10), and the framework was authored in v7 (line 831, also Round-5 = pre-Phase-5). The temporal sequence is: empirical result → framework retrofit → framework re-applied to *next* empirical result. Calling this "prospective prediction" overclaims by an order of magnitude. **Required fix**: either (i) reword §5.4 / Abstract / §1.5 / §8.1 to "post-hoc synthesis that unifies §5.2 and §6.4" and remove "predict-then-verify" framing, or (ii) provide concrete artifact-level evidence (commit hash of framework writeup *predating* the empirical result commit hash) that the prediction-then-verify ordering actually held. The latter is the higher bar and I see no evidence it can be met given v7/v8 changelog dates.

- **[CRIT-2] Abstract overclaims free-lunch passage relative to §6.2.3 self-correction.** Abstract (line 13): "본 논문이 제안하는 ***free-lunch*** 기준 ... 을 5개 비교 baseline ... 중 *유일하게* 통과". §6.2.3 itself (lines 376–378) reframes: "본 절의 정확한 reframing은: 'Δdf headline 효과는 PlotQA에서만 CI-strong이고 4 small-n cell에서는 점추정 부호 일관 + CI individually-inconclusive — 그러나 strict-free-lunch 의 *Δem(non-anchored arm) ≥ 0* clause ... 는 5/5 cell에서 multiplicity-corrected CI까지 sign-clean으로 통과 — 본 mitigation이 paper headline으로 권장되는 *robust* 신호는 b-arm em 회복이며, anchoring task의 df 감소는 sample-size 의존 strong-evidence cell (PlotQA) + 4 cell 점추정-일관-CI-borderline 으로 보고된다." The 4-clause free-lunch definition (line 380) requires `Δdf(anchoring task) < 0` AND `Δem(anchored) ≥ 0` AND `Δem(non-anchored) ≥ 0` AND `Δ(held-out) ≥ −0.5 pp` — by §6.2.3's own CI-augmented reading, the Δdf clause is multiplicity-clean only on PlotQA (1/5 datasets), not on the cross-evaluation as a whole. **The "free-lunch passed" claim conflates** (a) point-estimate sign consistency 5/5 with (b) CI-clean multiplicity-corrected 5/5. The abstract should mirror the body's reframe: "Δem(non-anchored) clause is the multiplicity-robust headline; Δdf clause is CI-clean only on PlotQA." Without this fix, the abstract is what readers will cite and remember, and it is at variance with what the body actually claims. Severity: this is the single biggest credibility hit for a methodology-stringent reviewer. (Note: the body section has been honestly updated; only the abstract framing lags.)

- **[CRIT-3] Paper is 834 lines and embeds a v3–v8 internal changelog at lines 831–833 — this is not a submittable conference paper.** Top venues (EMNLP / NeurIPS / ACL) accept 8-9 page papers (~3,500–4,500 words main text). The current draft Korean prose is roughly 4× the budget, and embeds: (a) version-history changelog at lines 831–833 (literally "v3 changes" / "v4 changes" / ... / "v8 changes" — 3 paragraphs of internal review-loop tracking that mention reviewer rounds, internal experiment IDs `P1-3`/`P1-6`/`P4-12`, canonical CSV path `_data/...`, and generator script names `scripts/build_e6_stage4_bootstrap_ci.py`); (b) inline references to internal evidence files in body prose (line 189: "출처 `docs/insights/L1-confidence-modulation-evidence.md` 2026-05-10 update"; line 188-189: "`cross_entropy`, length-invariant paper-clean default"; line 446: "본 라운드에서 동일 paired-bootstrap framework를 §6.2.3 Table 7 안에 직접 통합 (P1-3 close, 2026-05-10)"). These are all review-trail artefacts, not paper content. The paper has multiple footnote-tier hedges propagated into main body sections (§1.3 "본 E6 검증 chain은 단일 모델 `llava-onevision-qwen2-7b-ov` 위에서의 case study이며", §6.6 same hedge, §8.1 same hedge — three near-duplicate sentences). ## Statistical rigor audit

**Paired-bootstrap CI is now correctly reported for §6.2.3 Table 7 and §7 Table 9** — this was a critical V1 ask and is now addressed (v8 P1-3 close). The remaining gaps:

- **§4.2 Table 3 (a − m) gaps have no CIs** — MAJOR-5 above. 4 of the 11 reported gaps are sample-size-marginal (Qwen2.5-VL family on PlotQA n_wb=902/1,153, +1.0/+1.8 pp; InfoVQA/ChartQA on Main n_wb=403/211, +0.7/+1.4 pp). The "5/5 dataset (a − m) > 0" universality claim conflates point-estimate sign with CI-clean evidence.

- **§4.5 Table 4 ×12.7 ratio has no CI** — MAJOR-6 above. The ratio is computed from two point estimates (Instruct correct df=0.021, Thinking correct df=0.267) and reported to 3 sig figs. With underlying n_correct ≈ 249/385 and numeric_anchor denominator likely < 200 per arm, single-sample variance on the numerator can swing the ratio by ±50 %.

- **§4.6 Table 5 Bonferroni correction is k=84 (within-Thinking only)** — but the paper's framework prediction "anchor signal accumulates in late residual" effectively pre-registered a directional hypothesis on a specific layer band. The 84-cell sweep then *post-hoc* identifies 14 cells as Bonferroni-clean. This is a multiple-comparisons issue inside a multiple-comparisons correction: the *sweep itself* is the data-driven discovery, not the cell-by-cell test. **Suggested fix**: report which 14 cells were predicted by the framework *before* the sweep and what fraction of those framework-predicted cells then passed Bonferroni — that's the actual predictive power.

- **§6.2.3 Bonferroni-20 correction** (line 372) — 5 datasets × 4 metrics = 20 paired tests. Bonferroni-20 is conservative for paired bootstrap with shared sids but is the right family-wise floor. Honestly applied; Δem(b) 5/5 surviving 99.75 % CI is solid.

- **§7 Bonferroni-6 mention** (line 445) — applied post-hoc; HallusionBench and POPE conclusions survive. This is a competent statistical move.

Overall the *headline* statistics are now solid; the *secondary* tables (§4.2, §4.5) need the same treatment.

**Empirical anchor 1 (§6.4 LEACE +56 % backfire)**: post-hoc (per CRIT-1).
